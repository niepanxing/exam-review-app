一、CV二开定义与背景

  • CV二开定义：为满足各省市用户CV监控业务中长尾AI场景需求，视联网&AI公司打通算法标训平台、推理平台与应用，对省市二开需求进行支持
  • 三大痛点：
    • 属地烟囱式发展：各省市独立建设，重复造轮子
    • 研发交付链路不通：算法与应用脱节
    • CV生产周期长：从需求到交付周期漫长
  • 现状：视联网设备9000万+，CV算法覆盖率仅11%

CV二开三大业务平台

  • 视联网合作平台：申请管理入口
  • 能力开放平台：项目支撑
  • 视联网标品业务平台：云眼&看家

二、算法的训练、生产与入仓

星海标训工具

  • 低/零代码流程：数据标注 → 基础模型选择 → 超参数设置 → 算法训练 → 算法测评
  • 高代码流程：数据标注 → 模型搭建 → 资源接入 → 模型训练&评测

算法入仓流程

  • 自有算法入仓：算法注册 → 镜像注册 → 算法评测 → 算法上架（星河平台）
  • 第三方入仓：算法注册 → 镜像注册 → 算法评测（安全/精度/接口/性能/稳定性） → 算法上架
  • Pipeline封装：模型硬件适配 → Pipeline封装（检测+分类整合SDK） → 上架星河

算法边界

  • 零代码支持：目标检测属性识别
  • 高代码额外支持：语义分割OCR
  • 当前不支持：比对类

三、二开解决方案分类

  • 能力开放类应用（2B2G项目制）：通过API/组件/卡片进行深度开发
  • 标品业务类应用（2C按月订购）：BC类销售品对接

三大合作模式

  • 公有云二开：2元/路月算法加载
  • 边缘侧部署二开：硬件一次性结算
  • 云眼B/C类合作：2元/路月

成本构成

  • 算力成本 + 天翼视联服务成本 + AI算法研发成本 + 应用侧建设成本
  • 算力成本公式：每路算力成本 ≈ 硬件成本 / 总路数
  • 结算链路：云公司 → 天翼视联 → 省公司

四、积木式应用与BC类销售品

  • N+1+1架构：N个AI+应用、1个应用平台、1个开放平台
  • 支持零代码/低代码/高代码二开

BC类销售品

  • DICT单一客户走能力开放
  • 全省规模发展走BC类销售品对接
  • 准入标准：业务发展量（规模化可能性）、算法质量（P>80%, R>70%）、适配范围（至少省内）、算法性能（QPS>50）
  • 原则：不与A类大网销售品冲突；省B/C不纳入一点结算

典型案例

  • 上海垃圾分类：高代码+第三方入仓
  • 自贡智慧消防：零代码
  • 陕西智慧畜牧：积木应用
